# Code Review Skill

When reviewing code, follow this checklist:

1. Correctness: does the code do what it is supposed to do?
2. Edge cases: are missing values, duplicates, and invalid inputs handled?
3. Style: does it follow project conventions and stay readable?
4. Testability: are important tests missing?
5. Maintainability: is the code modular and easy to extend?

How to respond:
- Group findings by severity: high, medium, low.
- Be specific about what should change.
- Explain why it matters.
- Suggest a concrete fix when possible.
